# iPhone 短信转发快捷指令配置

本项目的短信验证码,通过 iPhone「快捷指令」自动化转发到服务器 Mac 的 webhook。配一次,之后全自动。

## 原理

```
iPhone 收到短信 → 快捷指令自动化触发 → 提取短信内容 → POST 到服务器 webhook → 服务缓存 → CLI 取码
```

## 前置

- iPhone 与服务器 Mac 在**同一 WiFi 局域网**
- 服务器 webhook 已启动:`pnpm webhook`(默认 `http://<服务器IP>:7788/sms`)
- 知道服务器 Mac 的局域网 IP(如 `192.168.1.x`,可用 `ifconfig | grep inet` 查)
  - 建议在路由器给服务器 Mac 固定 IP,或用 mDNS 主机名(如 `xxx.local`)

## 配置步骤

### 1. 新建自动化
iPhone「快捷指令」App → 底部「自动化」→ 右上角「+」/「新建个人自动化」→ 触发条件选**「信息」**。

### 2. 设置触发条件
- **发件人**:`任意`(或填 B站服务号,通常 `106` 开头)
- **信息包含**:可填 `验证码` 或 `哔哩哔哩` 过滤,减少误触发
- **立即运行**:✅ 打开(关闭"运行前询问")→ 实现免手动确认

### 3. 添加「获取 URL 内容」操作
点「添加操作」→ 搜**「获取 URL 内容」**(Get Contents of URL):

| 字段 | 值 |
|---|---|
| URL | `http://<服务器IP>:7788/sms` |
| 方法 | `POST` |
| 请求体(Body) | `JSON` |

JSON 字段(用快捷指令的变量插入):
```json
{
  "from": "<发件人变量>",
  "body": "<信息内容变量>",
  "received_at": "<当前日期变量>"
}
```

> 在 Body 里,点 `String` 切换为 JSON 三个键:`from` / `body` / `received_at`,值分别插入快捷指令的「发件人」「信息」「当前日期」魔法变量。

### 4. 测试
让 B站发一条验证码短信(或在「信息」里手动造一条含"验证码"的短信触发),确认服务器终端日志打印收到。

## webhook payload 格式

```json
{
  "from": "1069xxxxxxx",
  "body": "【哔哩哔哩】您的验证码为 123456,5 分钟内有效...",
  "received_at": "2026-07-15T23:00:00+08:00"
}
```

服务端只关心 `body`(用来正则提取验证码)和 `from`(可选过滤)。

## 备注 / 排错

- **iOS「信息」自动化触发器**对纯 SMS 的支持在不同版本有差异。若不触发:
  - 收紧"信息包含"关键词(如特定数字、`哔哩哔哩`)
  - 或改用 [Bark](https://apps.apple.com/us/app/bark-customed-push/id1403753865) / Scriptable 等做推送
- **服务器 IP 变化**时,需回到快捷指令更新 URL。
- webhook 默认端口 `7788`,可用 `pnpm webhook -- --port 8080` 改。
- 链路握手:CLI 在触发"获取验证码"前会先确认 webhook 可达,不通就不发短信(避免反复触发频率限制)。
